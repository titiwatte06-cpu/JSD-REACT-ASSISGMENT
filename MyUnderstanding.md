อธิบาย Logic ของโปรเจกต์ React นี้
ภาพรวมของโปรเจกต์
โปรเจกต์นี้เป็นเว็บแอปพลิเคชันที่สร้างด้วย React โดยใช้ React Router สำหรับการจัดการหน้าต่างๆ และ Tailwind CSS สำหรับการตกแต่ง มีทั้งหมด 4 ไฟล์หลักที่ทำงานร่วมกัน ได้แก่ Layout, Navbar, Hero, และ Owner

Layout และ Navbar — โครงกระดูกของแอป
เริ่มจากส่วนที่เป็นโครงสร้างหลักก่อน ไฟล์ Layout.jsx ทำหน้าที่เป็น "กรอบ" ของทุกหน้า โดยข้างในมีแค่ <Navbar /> ตามด้วย <Outlet /> ซึ่ง Outlet คือตัวบอกว่า "ตรงนี้แหละให้ React Router เอาเนื้อหาของแต่ละหน้ามาวาง" ผลลัพธ์คือ Navbar จะปรากฏอยู่ทุกหน้าตลอดเวลา ไม่ว่าผู้ใช้จะไปอยู่ที่ path ไหน
ส่วน Navbar.jsx ก็เรียบง่ายมาก มีแค่ Link สองอัน คือ Home (/) กับ Owner (/owner) ซึ่งใช้ <Link> จาก React Router แทนที่จะใช้ <a> ธรรมดา เพื่อให้การเปลี่ยนหน้าเป็นแบบ SPA ไม่มีการโหลดหน้าใหม่

Hero — หัวใจของแอป
ไฟล์นี้ซับซ้อนที่สุด เพราะมีทั้ง State หลายตัว, การดึงข้อมูลจาก API, และ Logic การแสดงผลแบบมีเงื่อนไข
การดึงข้อมูล: ตอนที่ Component นี้โหลดขึ้นมาครั้งแรก useEffect จะทำงานทันทีโดยไปดึงข้อมูลสมาชิกจาก MockAPI ซึ่งเมื่อได้ข้อมูลกลับมาก็จะเก็บไว้ใน State ชื่อ data เพื่อเอาไปแสดงในตาราง
ระบบสองโหมด (User และ Admin): ไอเดียหลักของหน้านี้คือมีปุ่มสองปุ่ม แต่ละปุ่มควบคุม State boolean ของตัวเอง คือ showUserTable และ showAdminTable ตาราง ข้อมูลสมาชิกจะแสดงก็ต่อเมื่อ showUserTable เป็น true เท่านั้น แต่ถ้าเปิดทั้งสองพร้อมกัน นั่นคือโหมด Admin จะมี Column "จัดการ" และฟอร์มเพิ่มสมาชิกปรากฏขึ้นมาเพิ่มเติม
การเพิ่มและลบสมาชิก: ฟังก์ชัน handleAdd จะรับค่าจาก input สามช่อง แล้วสร้าง object ใหม่ต่อท้าย array ที่มีอยู่ ส่วน handleDelete จะกรอง item ที่ตรงกับ id ที่ต้องการลบออก แล้ว map ใหม่เพื่อเรียง id ใหม่ให้ต่อเนื่อง ทั้งสองฟังก์ชันนี้แก้ไข state ฝั่ง client เท่านั้น ไม่ได้ส่งข้อมูลกลับไปที่ API
State ชื่อ home: ตัวนี้ทำหน้าที่เปลี่ยนข้อความ subtitle ใต้หัวข้อหลัก ให้สะท้อนว่าขณะนี้อยู่ในโหมดไหน

Owner — หน้าโปรไฟล์
หน้านี้เป็น Static Component ที่ไม่มี State หรือ Logic อะไรเลย แสดงแค่ชื่อเจ้าของ, กรอบรูปที่ใช้ SVG วาดเส้นทแยงมุมไขว้กัน และ Bio สั้นๆ เป็น Lorem Ipsum

สรุปภาพรวมการไหลของข้อมูล
Layout ห่อทุกอย่างไว้ Navbar นำทางระหว่างหน้า Hero จัดการ state และ data ทั้งหมดด้วยตัวเอง ส่วน Owner แค่แสดงผลอย่างเดียว โปรเจกต์นี้แสดงให้เห็นการใช้  -
